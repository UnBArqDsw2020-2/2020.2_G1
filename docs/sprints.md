# <center>Organização das Sprints
<br>
<div align="justify">
As sprints do projeto são planejadas ao final de cada semana e consolidadas em uma issue no repositório original. A oraganização consiste em atribuir um responsável por cada issue e a dupla que o acompanhará durante o desenvolvimento. 
<br><br>
Porém, as issues referentes à documentação possuem mais participantes responsáveis para assegurar que um mesmo documento será analisado por diferentes pontos de vista. As issues que forem classificadas como dívidas técnicas recebem o símbolo de "alerta" e são realocadas para sprints posteriores.
